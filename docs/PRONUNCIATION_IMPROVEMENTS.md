# 发音优化说明

## 📢 问题反馈

小朋友反馈：**读音不够标准**

## 🔍 问题分析

### 原因
1. **音频文件缺失** - 原本设计的 `/audio/*.mp3` 文件不存在
2. **依赖 Web Speech API** - 浏览器内置的 TTS 引擎质量参差不齐
3. **音标符号直接读** - 用英文字母拼读音标（如 /i:/ 读成 "ee"）不够准确

## ✅ 优化方案

### 已实施的优化（方案 3）

#### 1. 优化语音引擎选择
```javascript
// 优先级排序：
1. Google 美音 (Google US English) - 最自然
2. 其他美式英语
3. 英式英语
4. 任何英语
```

#### 2. 优化发音顺序
**之前：** 先读音标 → 停顿 → 读单词
**现在：** 先读单词（标准发音）→ 停顿 → 读音标（用对应单词）

**理由：**
- 单词是完整的英文，TTS 发音最准确
- 先用单词建立正确发音印象
- 音标用映射的示例词发音，更自然

#### 3. 优化语音参数
```javascript
rate: 0.85    // 稍慢一点，更清晰（原 1.0）
pitch: 1.0    // 正常音调
volume: 1.0   // 最大音量
pause: 500ms  // 音标和单词之间的暂停
```

#### 4. 优化音标映射
**之前：**
```javascript
'i:': 'ee',
'ɪ': 'ih'
```

**现在：**
```javascript
'i:': 'see',      // 用完整单词
'ɪ': 'sit',
'u:': 'blue',
'ʊ': 'book'
```

### 未来优化（方案 1 - 待实施）

**音频来源：**

3. 录制专业发音

**优点：**
- 发音最标准、专业
- 不依赖网络和 API
- 用户体验最佳

## 🎯 使用建议

### 对于小朋友
1. **先听单词** - 认真听完整的单词发音
2. **再听音标** - 注意音标在单词中的发音
3. **跟读练习** - 大声跟读，录音对比

### 对于家长
1. **检查音量** - 确保设备音量适中
2. **安静环境** - 在安静环境下学习效果更好
3. **多次练习** - 每个音标至少听 3 遍

### 修改的文件

### 关键代码
```javascript
// 优化后的播放顺序
async speakPhoneticAndExample(symbol, exampleWord) {
  // 1. 先读单词（最标准）
  await this.speakWord(exampleWord)
  
  // 2. 暂停 500ms
  await new Promise(resolve => setTimeout(resolve, 500))
  
  // 3. 再读音标（用映射的单词）
  await this.speak(symbol, exampleWord)
}
```
